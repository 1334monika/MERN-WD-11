Understanding CSS Units

CSS units are essential for defining the size and spacing of elements on a web page. They can be divided into two main categories: Absolute and Relative.

1. Absolute Units

Absolute units are fixed and do not change based on any other element or screen size. The most common absolute unit is the pixel (px).

px (Pixels)

  • A pixel is a single dot on the screen.
  • 1px is the same size regardless of the parent element's size.
  • They are useful for elements that you want to have a fixed size, such as borders or icons.
Example: 
.element {
  border: 1px solid black; /* A border that is always 1 pixel wide */
  margin: 10px;           /* A margin that is always 10 pixels */
}

2. Relative Units

Relative units are flexible and their size is determined by the size of another element, the font size, or the viewport size. They are crucial for creating responsive and accessible designs that adapt to different screen sizes and user preferences.

% (Percentage)

  • The percentage unit is relative to the same property of the parent element.
  • If you set an element's width to 50%, it will be half the width of its parent.
Example (from your original file): 
.parent {
  width: 100vw; /* Takes up the full browser width */
}

.child {
  width: 50%; /* This will be 50% of the parent's width */
  height: 50%; /* This will be 50% of the parent's height */
}

vw and vh (Viewport Width and Viewport Height)

  • These units are relative to the viewport, which is the visible area of the browser window.
  • 1vw = 1% of the viewport's width.
  • 1vh = 1% of the viewport's height.
  • These are great for creating full-screen sections or elements that should always be visible on the screen.
Example (from your original file): 
.parent {
  width: 100vw;  /* The element will be as wide as the browser window. */
  height: 100vh; /* The element will be as tall as the browser window. */
}

rem (Root em)

  • The rem unit is relative to the font-size of the root element (<html>).
  • This is extremely useful for accessibility. If a user changes their browser's default font size, all elements sized with rem will scale accordingly.
Example (from your original file): 
html {
  font-size: 32px; /* We set the root font size here. */
}

p {
  /* 
    1rem is equal to the root font size.
    In this case, 1rem = 32px.
    If we changed the html font-size to 16px, this paragraph's font-size would become 16px.
  */
  font-size: 1rem; 
}

em

  • The em unit is relative to the font-size of the parent element.
  • If a parent has font-size: 16px, then 1em for a child element will be 16px, and 2em will be 32px.
  • This can be tricky because em units can compound. If you have nested elements each with font-size: 1.2em, the text will get progressively larger.
Example: 
.container {
  font-size: 20px;
}

.container p {
  /* This will be 1.5 * 20px = 30px */
  font-size: 1.5em; 
}

Summary and Best Practices

  • For Fonts and Spacing: Use rem for most of your typography and spacing (margin, padding). This ensures your layout scales predictably and respects the user's browser settings for font size, which is great for accessibility.
  • For Layouts: Use % when you want an element's size to be relative to its container (e.g., columns in a grid).
  • For Full-Screen Elements: Use vw and vh to size elements relative to the browser window itself.
  • For Fixed Sizes: Use px for things you don't want to scale, like borders or very specific, small design elements.
